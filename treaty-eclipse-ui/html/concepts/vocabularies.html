Vocabularies

The resource types and relationships are defined in a contract vocabulary. The vocabulary is not static but dynamic. It is assembled at runtime from plugins that contribute vocabularies. The respective extension point in the Treaty for Eclipse plugin is net.java.treaty.eclipse.vocabulary. Plugins extending this extension point have to provide the following:

  1. A list of contributed resource types

  2. A list of contributed relationship and property types

  3. A method that can be used to find out whether two resources instantiate a contributed relationship


The last part defines the semantics of the relationship. For the relationships used in the clock example, plugins use tools such as the Java reflection API, JUnit and validating XML parsers to compute this information.

Extensions must provide an implementation of the net.java.treaty.ContractVocabulary interface and describe the contributed vocabulary elements through a ontology (OWL file). The vocabulary view collects all these vocabulary contributions, merges them into one ontology and displays the elements in an Eclipse view. The merged ontology is also available to plugins (for instance, to take advantage of ontology reasoning) through the singleton class net.java.treaty.eclipse.Vocabulary. The ontology is available as unparsed text (string), and as parsed in memory representation as instance of org.semanticweb.owl.model.OWLOntology.
